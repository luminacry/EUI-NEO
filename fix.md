# 修复记录

本文档汇总本轮界面、渲染、模糊、交互性能和主题适配中出现过的问题、定位结果与处理方式，便于后续继续维护时避免回归。

## 1. 交互时 GPU 占用偏高

### 现象

- 鼠标 hover 按钮
- 打开下拉框
- 输入文字
- 拖动滑条

以上操作在局部交互时会把 GPU 占用拉高到 20% 甚至 30%。

### 定位结果

- 交互虽然是局部变化，但如果脏区覆盖不准确，容易把更多区域卷入重绘。
- 带模糊和阴影的矩形如果仍按原始盒子范围绘制，实际显示区域与脏区区域不一致，会导致重复绘制或残影。
- 模糊缓存复用在当前实现下不稳定，复用旧结果容易产生错误内容，反而增加排查和修复成本。

### 处理方式

- 保留局部脏区重绘机制，避免普通交互直接全屏重绘。
- `DrawRect` 改为按扩展后的真实边界参与绘制和脏区计算，而不是只按原始 box。
- 模糊缓存复用当前直接关闭，优先保证交互稳定性和显示正确性。
- 主题切换、窗口尺寸变化、模糊参数变化时，显式失效 backdrop 和相关缓存。

### 当前约束

- 模糊矩形、带阴影矩形、带 transform 的矩形，脏区必须按最终显示边界计算。
- 在没有完整样式等价判断前，不允许重新开启模糊缓存复用。

## 2. 模糊面板出现直角半透明边

### 现象

- 圆角模糊面板四角出现矩形半透明边。
- 面板看起来像“模糊外面又套了一层透明方块”。

### 定位结果

- 模糊采样、片元裁剪和绘制边界没有完全对齐。
- 圆角矩形带 blur 时，如果边界仍按原始矩形处理，就会在角落保留不该存在的透明像素。
- 局部重绘或缓存复用时，错误边界会放大这个问题。

### 处理方式

- 统一 `DrawRect` 的边界测量逻辑，使用真实外扩边界。
- 片元阶段改为基于局部形状坐标计算圆角裁剪。
- 模糊采样改为按像素半径采样，并对 UV 做 clamp，避免边缘取样脏出界。
- 模糊缓存复用禁用，优先消除边角残留。

### 当前约束

- 阴影不能用“额外半透明方块”假装。
- 模糊与阴影必须走同一套圆角裁剪逻辑。

## 3. 圆形或圆角模糊看起来像花瓣

### 现象

- 圆形模糊区域出现放射状纹理。
- 视觉上像“花瓣”或者非均匀晕开。

### 定位结果

- 模糊半径如果按归一化 UV 直接放大，横纵比例和采样方向容易失真。
- 在圆角、旋转、缩放叠加时，这种失真更明显。

### 处理方式

- 模糊半径改为像素半径计算。
- 采样分布改为围绕当前像素位置采样，而不是直接放大归一化坐标。
- 保持模糊仍然来自 backdrop，不改变原有视觉方向。

## 4. 控件出现半透明矩形边框

### 现象

- 下拉框、侧边栏、模糊面板、自绘卡片等区域曾出现半透明矩形边。
- 内置控件和自绘控件圆角、阴影、边界风格不统一。

### 定位结果

- 一部分边框不是组件真实样式，而是为了“补阴影”或“补高光”额外叠出来的矩形。
- 自绘页面直接画了额外底板、内框或分隔层，和组件自身的圆角裁剪不一致。

### 处理方式

- 删除伪阴影、伪边框、伪遮罩矩形。
- 阴影统一通过 `shadowBlur / shadowOffset / shadowColor` 实现。
- 自绘面板和内置控件尽量复用一致的圆角和颜色 token。

### 当前约束

- 不允许再用半透明直角矩形冒充阴影。
- 新增自绘控件时，先保证圆角裁剪正确，再谈视觉增强。

## 5. 下拉框按钮与展开列表之间有分界线

### 现象

- ComboBox 关闭态按钮与展开列表之间能看到一条分界线。
- hover 某些 item 时顶部接缝更明显。

### 定位结果

- 主按钮和下拉列表的绘制顺序会直接决定接缝是否可见。
- 旧实现里存在额外 top line / focus line 风格绘制，放大了这条分界线。
- 列表和按钮边界恰好贴齐时，1px 接缝在浅色主题下非常明显。

### 处理方式

- 下拉列表先画，主按钮后画。
- 列表顶部向上覆盖 1px，消除按钮与列表之间的接缝。
- 移除多余的顶部分隔线与伪 focus 线。
- item hover 背景仅在实际可见区域内绘制。

### 当前约束

- 下拉类组件不允许再额外画与主体圆角不一致的线条。
- 按钮体和弹层接缝必须通过重叠或统一边界解决，不能靠额外描边修补。

## 6. 白天模式适配不完整，切换后刷新不及时

### 现象

- 侧边栏仍像夜间模式。
- Animation 页面颜色不跟随主题变化。
- 切换主题后要过一会儿才刷新完全。

### 定位结果

- 自绘页面和自绘导航没有全部接入主题同步。
- 主题变化后没有统一失效 blur/backdrop 与页面自绘区域。

### 处理方式

- 在主页面集中执行 `ApplyShellTheme()`。
- 主题切换后显式调用 `animationPage.OnThemeChanged()`。
- 同时调用 `Renderer::InvalidateBackdrop()` 与 `Renderer::InvalidateAll()`，确保模糊和内容一起刷新。

### 当前约束

- 任何自绘页面都必须提供 `OnThemeChanged()` 或等价入口。
- 主题切换不能只改颜色变量，必须同时触发重绘失效。

## 7. Animation 页面 hover 遮罩过重，文字被示例块挡住

### 现象

- 卡片 hover 时整块被一层底色罩住。
- 右侧示例块压住说明文字。
- 页面视觉重点被 hover 底板抢走。

### 定位结果

- 卡片 hover 直接改整个 panel 的填充色，造成明显“罩层感”。
- 说明文字和右侧示例区共用同一水平空间，没有留足内容边距。
- 示例块底板又叠了一层浅色矩形，进一步压迫文字空间。

### 处理方式

- 卡片 hover 只保留位移与示例块动画，不再修改整块 panel 底色。
- 说明文字改成更短的两行，避免横向冲突。
- 收紧并重新定位右侧示例块。
- 删除示例块外侧额外底板。
- 侧边栏 hover 底板同步移除，仅保留图标亮度反馈和选中态指示。

### 当前约束

- 交互反馈优先用位移、透明度、缩放，不要优先加整块遮罩。
- 卡片内如果右侧存在预览区，文字布局必须先为预览区留出固定空间。

## 8. 字体未按预期生效

### 现象

- 页面字体一度没有走项目字体，表现接近系统 fallback。

### 定位结果

- 字体加载依赖运行目录，路径不稳定时会直接落入系统字体。

### 处理方式

- 主程序优先尝试加载项目内字体。
- 项目字体失败时，再回退到系统字体。
- 图标字体单独加载，避免图标和正文字体相互污染。

### 当前约束

- 新页面不要默认假设运行目录固定。
- 新增字体时必须提供项目路径优先、系统字体兜底的加载顺序。

## 9. 阴影效果被错误替换成透明遮罩

### 现象

- 视觉上想要的是阴影，但实际看到的是半透明直角块。

### 定位结果

- 阴影和遮罩是两种完全不同的视觉语义。
- 用半透明矩形叠层替代阴影，会直接破坏圆角边缘和层级关系。

### 处理方式

- 恢复阴影语义，使用真实阴影参数绘制。
- 去掉拿透明底板冒充阴影的写法。

### 当前约束

- 阴影必须只来自 shadow 参数。
- 不允许用额外矩形叠加来“模拟阴影”。

## 10. 本轮最终保留的维护原则

### 渲染层

- `DrawRect` 必须以最终可见边界参与绘制、裁剪与脏区计算。
- blur 相关优化以“先正确、后缓存”为原则。
- backdrop 失效必须显式可控。

### 组件层

- 下拉框、输入框、按钮这类高频交互控件，必须精确标记 dirty 区域。
- 自绘控件与内置控件要统一圆角、阴影和主题 token。
- 不要为了补视觉效果再叠加额外透明矩形。

### 页面层

- 自绘页面必须在主题变化时同步刷新。
- hover 反馈优先用 transform、alpha、scale，不优先用整块罩层。
- 卡片布局先保证内容不遮挡，再叠加动效。
